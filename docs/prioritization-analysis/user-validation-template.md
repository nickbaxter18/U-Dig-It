# User Validation Template

**Generated**: 2025-01-23
**Purpose**: Template for validating priorities with actual admin users

---

## Admin User Interview Guide

### Interview Setup

**Participants**: 2-3 admin users
**Duration**: 30-45 minutes per user
**Format**: One-on-one interviews or small group discussion
**Location**: In-person or video call

---

## Interview Questions

### 1. Current Workflow Questions

**Q1: What are your most common daily tasks?**
- Purpose: Understand current workflows
- Follow-up: Which tasks take the most time?
- Expected: Logistics, payments, maintenance, customer management

**Q2: What manual processes do you perform regularly?**
- Purpose: Identify pain points
- Follow-up: How long do these take?
- Expected: Spreadsheet tracking, manual data entry, phone calls

**Q3: What workarounds are you currently using?**
- Purpose: Understand current solutions
- Follow-up: How effective are these workarounds?
- Expected: Direct database access, spreadsheets, email/phone

---

### 2. Feature Priority Questions

**Q4: Which features would save you the most time?**
- Present: Prioritized feature list
- Ask: Rank top 5 features by time savings
- Expected: Logistics, maintenance alerts, admin user management

**Q5: What manual processes are most painful?**
- Present: List of manual processes from analysis
- Ask: Rate pain level (1-10) for each
- Expected: Admin user management, payment reconciliation, logistics

**Q6: Which features do you use via direct API calls or database access?**
- Purpose: Identify features that need UI
- Follow-up: How often do you do this?
- Expected: Admin user management, manual data entry

**Q7: What features have you requested before?**
- Purpose: Validate user demand
- Follow-up: How long have you been waiting?
- Expected: Admin user management, export functionality

**Q8: What features would you never use?**
- Purpose: Identify features to deprioritize
- Follow-up: Why wouldn't you use them?
- Expected: Telematics, insurance features (if infrequent)

---

### 3. Feature Validation Questions

**Q9: Review Top 10 Features**
- Present: Feature Priority Matrix (top 10)
- Ask: Do you agree with the priority order?
- Follow-up: What would you change?

**Q10: Are there any missing critical features?**
- Purpose: Identify gaps in analysis
- Follow-up: What would you add?
- Expected: May identify additional features

**Q11: Which features are "nice-to-have" vs "must-have"?**
- Present: Tier 1 (Must-Have) vs Tier 2 (Should-Have)
- Ask: Do you agree with the categorization?
- Follow-up: What would you move between tiers?

---

### 4. Implementation Feedback Questions

**Q12: If you could only have 5 features, which would they be?**
- Purpose: Validate top priorities
- Expected: Admin user management, maintenance alerts, logistics, payments

**Q13: What's the minimum feature set you need to do your job effectively?**
- Purpose: Identify MVP features
- Expected: Core admin functions, basic reporting

**Q14: Are there any features that would unlock other capabilities?**
- Purpose: Identify dependencies
- Expected: Admin user management unlocks other admin features

---

## Feature Validation Checklist

For each feature in the priority list, ask:

- [ ] **Would you use this feature?** (Yes/No)
- [ ] **How often would you use it?** (Daily/Weekly/Monthly/Rarely)
- [ ] **How much time would it save?** (Hours per week/month)
- [ ] **What's your pain level without it?** (1-10 scale)
- [ ] **Is this a "must-have" or "nice-to-have"?** (Must/Should/Nice/Defer)
